# DUCP Proposals

Substantive changes to the protocol are proposed here, in the open, before they
enter the specification. This keeps the protocol's evolution transparent and
reviewable.

## When to open a proposal

Open a proposal for anything that changes the protocol's design — the unit,
verification, economics, Standing, governance, or the locked identity described in
§12 of the white paper. Small editorial fixes don't need one; open a pull request
instead (see [`../CONTRIBUTING.md`](../CONTRIBUTING.md)).

## Process

1. Open an issue labelled `proposal` summarizing the idea.
2. Add a proposal document here as `NNNN-short-title.md`, using the template below.
3. Discussion happens in the open. During the pre-1.0 phase the maintainer makes the
   final decision (see [`../GOVERNANCE.md`](../GOVERNANCE.md)); accepted proposals are
   versioned per [`../CHANGELOG.md`](../CHANGELOG.md).

## Template

```
# DP-NNNN: Title

- Status:   Draft | Under review | Accepted | Rejected | Superseded
- Author(s):
- Created:

## Summary
One paragraph.

## Motivation
The problem being solved.

## Specification
The change, concretely — and which white-paper sections and mechanisms it touches.

## Alternatives considered
And why this one.

## Open questions / risks
```
